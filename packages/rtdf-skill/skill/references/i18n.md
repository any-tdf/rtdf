# RTDF Internationalization

RTDF components use `ConfigProvider` for locale text. The default locale is Simplified Chinese.

## Configure Locale

Wrap the application root or a subtree with `ConfigProvider`:

```tsx
import { ConfigProvider } from 'rtdf';
import { en_US } from 'rtdf/lang';

const App = () => (
	<ConfigProvider locale={en_US}>
		<Routes />
	</ConfigProvider>
);

export default App;
```

All descendant RTDF components read the current locale. For scoped wording, wrap a subtree with another `ConfigProvider`.

## ConfigProvider Props

| Name | Type | Meaning |
| --- | --- | --- |
| `locale` | `LangProps` | Component internationalization text config |
| `theme` | `SwitchThemeInput` | Theme config, applied through RTDF theme helpers |
| `mode` | `'primary' \| 'dark'` | Light or dark mode config |
| `iconPath` | `string` | Global SVG symbol path used by `Icon` |

## Locale Presets

Locale presets are exported from `rtdf/lang`.

When configuring an application:

- Import an existing preset from `rtdf/lang` when it fits the project.
- Create an app-local locale object when the product needs custom wording.
- Keep locale keys consistent with an existing preset shape.

## Current Locale Coverage

RTDF includes many locale presets, including `en_US`, `zh_CN`, `zh_TW`, `ja_JP`, `ko_KR`, `es_ES`, `fr_FR`, `de_DE`, `it_IT`, `ru_RU`, and others. Inspect the installed `rtdf/lang` package exports for the exact current list.
